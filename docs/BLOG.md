# 我拿 Cursor 在 3 天内做了一个油管视频翻译软件

起因是这样的：

本人在使用油管自带翻译时经常会遇日语翻译时精度远低于英语视频的情况，而我经常看的管人和游戏区常常会因为专有名词而导致翻译的一塌糊涂，这时候期待烤肉man翻译时长达到几小时的直播回放显然是强人所难了。

所以我想：**能不能自己做一个「懂游戏黑话、懂管人梗」的翻译工具？** 不一定要等熟肉，至少先把字幕啃下来。

三天后，我有了这个项目。

---

## 这玩意儿到底能干啥

说白了就四件事，都在一个网页里搞定：

1. **贴一句日语** —— 比如直播里突然蹦出来的「このゲームのデバフがエグい」，点翻译，边看边出中文。
2. **丢一个 YouTube 链接** —— 自动扒 YouTube 官方/CC 字幕，边翻边在页面上预览。
3. **下载中文字幕** —— 翻完直接下 `{视频ID}_zh.srt`，时间轴跟原版对齐，可以丢进播放器当外挂字幕。
4. **上传本地 SRT/TXT** —— 手里已有字幕文件的也能批量翻。

默认界面很干净，只给你看译文；要是好奇 AI 怎么想的，勾上「显示 Agent 思考过程」，能看到它什么时候去查词库、查到了啥。

---

## 为什么不用「普通机翻」

油管自动字幕 + 浏览器翻译，对付日常 vlog 还行。一到游戏实况和 VTuber 杂谈就露馅：

| 你听到的 | 字面翻容易变成 | 实际想表达的意思 |
|----------|----------------|------------------|
| デバフ | 「缺点」？ | 减益、负面状态 |
| エグい | 「恶劣」？ | 强得离谱、离谱地狠 |
| ワンパン | 「一拳」？ | 一击秒杀 |
| 天井 | 「天花板」？ | 抽卡保底 |

这些词不是「翻错一个字」的问题，是**压根不在通用模型的常识里**。所以我没走「一个大模型硬翻到底」的路子，而是让它**拿不准就去查自己的术语本**。

---

## 核心思路：会查词典的翻译 Agent

可以把它想成一个有点强迫症的字幕组新人：

1. 接到一句日语。
2. 发现里面有黑话、缩写、梗 → **必须**调用 `search_term_dict` 去术语库里搜。
3. 把搜到的「推荐译法 + 含义 + 易错提醒」揉进上下文，再输出中文。

术语库按场景分了四本「小词典」：

- `gaming` —— 战斗、抽卡、版本环境词
- `vtuber` —— 管人圈常用说法
- `cooking` —— 料理实况（顺手做的，以后还能扩）
- `general` —— 兜底

更贴心的一点是：你贴 **YouTube 链接** 时，程序会读视频的标题、简介、标签、频道名，猜这是游戏区还是管人区，**只翻对应那本词典**，减少「料理视频里查到原神词条」这种乌龙。单句翻译和上传文件则默认查全库。

技术栈写给人看就是：前端 Node.js 做页面和流式转发，后端 Python + FastAPI + LangChain 管 Agent，词库用 ChromaDB 做检索，大模型接的是 DeepSeek（OpenAI 兼容接口）。

---

## 三天里 Cursor 帮了我什么

说实话，**三天能跑起来，Cursor 占了大半功劳**。我不是从零手写每一行，而是跟 AI 结对把坑一个个填平：

- **版本踩坑**：一开始 LangChain 版本和 `create_tool_calling_agent` 对不上，AI 直接指出要锁到 0.3.x，省了我半天瞎试。
- **油管玄学**：国内访问 YouTube 要代理；`RequestBlocked` 要回退 `yt-dlp`；`Sign in to confirm` 要从浏览器导 cookies——这些全是真机调试时冒出来的，AI 帮着改配置、改抓取逻辑，而不是让我自己啃文档啃到半夜。
- **产品细节**：术语从代码里硬编码改成 JSON 维护；思考过程默认折叠；长视频字幕按块翻译不截断——很多是聊着聊着「要不这样更顺手」就改掉了。
- **怎么知道翻得好不好**：还顺手写了 `eval/llm_judge.py`，20 条术语用例自动跑一遍，让另一个 LLM 当裁判打分，免得全凭感觉说「好像还行」。

我负责定方向、验收效果、决定「管人区必须能翻 凸待ち 这种梗」；Cursor 负责实现、排错、补文档。这分工挺像带了一个不会累的实习生。

## 用起来长什么样

Windows 上双击 `start.bat` 或跑 `.\start.ps1`，脚本会帮你装依赖、起后端（8000）和前端（3000），浏览器打开 http://localhost:3000 就行。

要准备的东西：

- Python 3.10+、Node.js
- `.env` 里填 `OPENAI_API_KEY`（我用的 DeepSeek）
- 国内访问 YouTube 一般需要 `YOUTUBE_PROXY`，有时还要 `YOUTUBE_COOKIES_FROM_BROWSER=firefox` 这类配置

**试一句**：输入 `このゲームのデバフがエグい`，正常应该能看到 Agent 去查「デバフ」「エグい」，译文里出现减益、强得离谱之类的人话，而不是「这个游戏的缺点很恶劣」。

**试一条链接**：选有官方字幕的视频，流式预览；满意再点「下载翻译字幕 (SRT)」。

## 还没完美的地方

诚实说局限：

- **长直播仍然慢**：字幕是按块喂给 Agent 的，几小时回放能翻，但要等，别和熟肉组比速度。
- **没有视频画面**：只抓字幕轨道，不下载整片视频。
- **词库要持续养**：新游戏、新梗得往 `backend/app/data/terms/*.json` 里加，不然还是会懵。
- **日语检索一般**：ChromaDB 默认 embedding 对日文语义不是最优，以后可能换多语言向量模型。

后续想做的：翻译 Agent + 校对 Agent 双阶段、超长视频分段并发、词库再扩一批管人/Event 专用词。

---

## 写在最后

这个项目起因很私人：**我就是想舒服地看日语游戏和管人视频，又不想被烂翻译折磨。**

三天 + Cursor，搭出了一个「会查黑话词典」的油管字幕翻译小工具。它替代不了专业烤肉组的感情和润色，但对付「官方字幕有了、翻译却像机翻」的日常观看，已经够用。

如果你也常被 `デバフ` 翻成「缺点」、被 `エグい` 翻成「恶劣」折磨，可以 clone 仓库按 README 跑起来试试。有问题或想补词条，欢迎提 issue——词库这东西，本来就是越用越厚的。
